**Agno turns agents into production software.**

Build agents in any framework. Run them as services. Ship to real users.

<CodeGroup>

```python Agno SDK
from agno.os import AgentOS
from agno.agent import Agent
from agno.db.sqlite import SqliteDb
from agno.tools.workspace import WorkspaceTools

agent = Agent(
    name="Agno Agent",
    model="openai:gpt-5.4",
    tools=[
        WorkspaceTools(
            base_dir=".",
            allowed_tools=["read_file", "list_files", "search_content"],
            confirm_tools=["write_file", "edit_file", "delete_file", "run_command"],
        )
    ],
)

agent_os = AgentOS(
    agents=[agent],
    tracing=True,
    db=SqliteDb(db_file="tmp/agentos.db"),
)
app = agent_os.get_app()

if __name__ == "__main__":
    agent_os.serve(app="agno_agent:app", reload=True)
```

```python LangGraph
from agno.agents.langgraph import LangGraphAgent
from agno.db.sqlite import SqliteDb
from agno.os import AgentOS
from langchain_openai import ChatOpenAI
from langgraph.graph import MessagesState, StateGraph

def chatbot(state: MessagesState):
    return {"messages": [ChatOpenAI(model="gpt-5.4").invoke(state["messages"])]}

graph = StateGraph(MessagesState)
graph.add_node("chatbot", chatbot)
graph.set_entry_point("chatbot")
compiled = graph.compile()

agent = LangGraphAgent(
    name="LangGraph Chatbot",
    graph=compiled,
)

agent_os = AgentOS(
    agents=[agent],
    tracing=True,
    db=SqliteDb(db_file="tmp/agentos.db"),
)
app = agent_os.get_app()

if __name__ == "__main__":
    agent_os.serve(app="langgraph_agent:app", reload=True)
```

```python DSPy
import dspy
from agno.agents.dspy import DSPyAgent
from agno.db.sqlite import SqliteDb
from agno.os import AgentOS

dspy.configure(lm=dspy.LM("openai/gpt-5.4"))

agent = DSPyAgent(
    name="DSPy Assistant",
```

Three imports, ten lines. You now have a FastAPI service with 50+ endpoints, persisted sessions, tracing, scheduling and RBAC. Wrap it in a container and deploy.

## Three layers, one architecture

| Layer | Use it to | What it does |
|-------|------|--------------|
| **SDK** | Build | Define agents, teams, and workflows with memory, knowledge, guardrails, and 100+ integrations. |
| **Runtime** | Run | AgentOS is a secure, stateless FastAPI backend with sessions, scheduling, and interfaces. |
| **Control Plane** | Manage | Monitor and operate your system from the [AgentOS UI](https://os.agno.com). |

## What AgentOS gives you

- [**Production API.**](/runtime/serve-as-api) 50+ endpoints with SSE and websockets to build your product on.
- [**Storage.**](/runtime/storage) Store sessions, memory, knowledge, and traces in your database.
- [**Context.**](/runtime/context) Navigate live context across Slack, Drive, MCP, and custom sources.
- [**Human approval.**](/runtime/human-approval) Pause runs for user confirmation, admin approval, or external execution.
- [**Observability.**](/runtime/observability) OpenTelemetry tracing, run history, and audit logs out of the box.
- [**Security & auth.**](/runtime/security-and-auth) JWT-based RBAC and multi-user, multi-tenant isolation.
- [**Interfaces.**](/runtime/interfaces) Slack, Telegram, WhatsApp, Discord, AG-UI, or your own custom interface.
- [**Scheduling.**](/runtime/scheduling) Cron-based scheduling and background jobs without any external infra.
- [**Deploy.**](/runtime/deploy) Docker, Railway, AWS, GCP. Any container host works.
