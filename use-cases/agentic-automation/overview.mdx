The gap between a chat agent and an automation agent is action. A chat agent is reactive. An automation agent is proactive: it reviews the PR, posts the report, and syncs on its own.

Agno gives you the tools to automate any repeatable body of work. It puts a boundary around the irreversible parts with human-in-the-loop and approval flows. Agno also comes with an in-process scheduler, so you don't need external infrastructure.

```python
from agno.agent import Agent
from agno.db.postgres import PostgresDb
from agno.models.openai import OpenAIResponses
from agno.os import AgentOS
from agno.tools import tool

db = PostgresDb(db_url="postgresql+psycopg://ai:ai@localhost:5532/ai")


@tool(requires_confirmation=True)
def close_issue(number: int, comment: str) -> str:
    """Close a GitHub issue. Pauses for human approval before it runs."""
    ...


agent = Agent(
    name="Ops Agent",
    model=OpenAIResponses(id="gpt-5.5"),
    db=db,
    tools=[close_issue],
    instructions="Triage stale issues. Explain every action before you take it.",
)

agent_os = AgentOS(agents=[agent], db=db, scheduler=True)
app = agent_os.get_app()
```

A tool that does real work, a confirmation gate on the irreversible step, and a scheduler, all served by AgentOS. Swap the tools and the same shape automates incident response, release management, or on-call triage.
