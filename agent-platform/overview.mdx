Every team building agents builds the same system from scratch:

- A server to run the agents (in batch, streaming, or background mode).
- A database for sessions, runs, traces, and memory.
- Auth and RBAC, validated via JWT on every request.

An agent platform handles the plumbing once so you can build agents and workflows without worrying about security, infrastructure or plumbing.

This guide shows you how to build an agent platform managed entirely by coding agents like Claude Code. Create new agents, improve them, run evals, fix drift -- all using claude code.

## Anatomy of an Agent Platform

Your agent platform is responsible for:

- Running agents in production (streaming, sessions, scheduling, RBAC).
- Storing what your agents read and write.
- Connecting agents to external tools and services.
- Exposing agents through Slack, Discord, or your own UI.
- Hosting it all on real infrastructure.

It brings these five parts together:

| Layer              | Responsibility                                                               |
| ------------------ | ---------------------------------------------------------------------------- |
| **Runtime**        | The service that runs agents. SSE streaming, sessions, scheduling, RBAC.     |
| **Storage**        | Sessions, memory, knowledge, traces, eval history. One database.             |
| **Connectors**     | Tools, MCP servers, APIs, CLIs the agents reach out to.                      |
| **Interfaces**     | Slack, Discord, Telegram, custom UIs. One place to resolve user identity.    |
| **Infrastructure** | Where it all runs. Docker locally, Railway (or any container host) for prod. |
