# Build docs/examples to match cookbook 1:1

## Objective

Build and sync `docs/examples/` so every section maps 1:1 to the `cookbook/` directory structure. Each cookbook subdirectory becomes a collapsible group in the sidebar. **Every single `.py` file in every cookbook directory gets its own dedicated MDX page.** No skipping files, no combining multiple files into one page, no summarizing. If a cookbook directory has 20 `.py` files, that section gets 20 pages.

NOTE: Always read the README in that section to help with the file order.

## Overview page

The examples/overview.mdx is the entrypoint to the examples documentation, we will update it separately.

## Example: Quickstart Section

Start with the Quickstart section, it maps `cookbook/00_quickstart/` that looks like this:

**Cookbook files:**
```
cookbook/00_quickstart/

  agent_with_storage.py
  ...13 total .py files
```

1. Read the cookbook/00_quickstart/README.md identify file order.

2. Update the **docs.json structure** to match (nested collapsible group):

```json
{
  "group": "Examples",
  "pages": [
    "examples/overview",
    {
      "group": "Quickstart",
      "pages": [
        "examples/quickstart/overview",
        "examples/quickstart/agent-with-tools",
        "examples/quickstart/agent-with-structured-output",
        "...1 page per .py file"
      ]
    }
  ]
}
```

3. Create MDX pages.

**MDX page structure** (each page has the full Python code inlined):

```mdx

---

\```

## Implementation Instructions

For each section:

### Step 1: Scan the cookbook directory

```bash
# List all .py files (excluding __init__.py)
find cookbook/<section>/ -name "*.py" ! -name "__init__.py" | sort
```

**Every file must be considered to become a page. No exceptions.**

### Step 2: Read the README in that folder

### Step 3: Create MDX pages

For each `.py` file that deserves to become a page, create an MDX page at the corresponding path under `docs/examples/`.

**Naming convention:**
- Cookbook: `cookbook/02_agents/04_tools/tool_choice.py`
- MDX: `docs/examples/agents/tools/tool-choice.mdx`
- Rule: Strip number prefixes from directories, convert underscores to hyphens.
  - Preserve a leading underscore in filenames (e.g., `_agents.py` → `_agents.mdx`) so special script names remain unambiguous.

**Page template:**
```mdx
---
title: <Human readable title from the Python file's docstring or filename>
description: <One-line description of the file.>
---

<Small intro paragraph from the docstring>

\```python
<FULL contents of the .py file, pasted inline>
\```

## Critical Rule

**Every single `.py` file (excluding `__init__.py`) in every cookbook directory must be considered to have its own dedicated MDX page.** Do not skip files. Do not combine multiple files into one page. Do not summarize. The mapping is strictly 1:1: one cookbook `.py` file = one MDX page with the full code inlined.

## Style Rules

- No emojis
- No "Learn how to..." intros - just state what it does
- Code-first: lead with the code block, then key concepts table
- Tables over prose
- Keep descriptions under 1 line
- Title should match the cookbook file's purpose, not the filename
- Use the Python file's docstring for the description when available
